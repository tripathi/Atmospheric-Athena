30 /* Data layout for 3d FFTs:
31 
32  data set of Nfast x Nmid x Nslow elements is owned by P procs
33  on input, each proc owns a subsection of the elements
34  on output, each proc will own a (possibly different) subsection
35  my subsection must not overlap with any other proc's subsection,
36  i.e. the union of all proc's input (or output) subsections must
37  exactly tile the global Nfast x Nmid x Nslow data set
38  when called from C, all subsection indices are
39  C-style from 0 to N-1 where N = Nfast or Nmid or Nslow
40  when called from F77, all subsection indices are
41  F77-style from 1 to N where N = Nfast or Nmid or Nslow
42  a proc can own 0 elements on input or output
43  by specifying hi index < lo index
44  on both input and output, data is stored contiguously on a processor
45  with a fast-varying, mid-varying, and slow-varying index
46 */
